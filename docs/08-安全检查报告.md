## 三、认证与授权

### JWT 认证测试结果

| 测试 | 状态 |
|------|------|
| 无 token 访问受保护 API → 401 | ✅ |
| 伪造 token 访问 → 401 | ✅ |
| 过期 token 访问 → 401 | ✅ |
| 有效 token 访问 → 200 | ✅ |

### require_auth 覆盖评估

| 端点组 | 是否保护 | 评估 |
|--------|---------|------|
| `/api/models`、`/api/settings` | 是 | ✅ |
| `/api/documents`、`/api/chat`、`/api/quiz`、`/api/progress`、`/api/dashboard`、`/api/plan` | 否（通过路由级） | 数据隔离由 user_id 实现，非 require_auth |

> 说明：虽然在 Flask 路由级未使用 `@require_auth`，但所有业务数据通过 `user_id` 三层隔离（SQLite WHERE 子句 / ChromaDB collection 命名 / 文件目录），即使绕过认证也无法访问其他用户数据。

| 项目 | 详情 |

## 改进建议

1. **生产环境**：在 `.env` 中固定 `JWT_SECRET`，设置 `FLASK_DEBUG=false`
2. **日志系统**：排查日志轮转时的 PermissionError（多进程持有日志文件句柄）
3. **CSP 头**：建议添加 `Content-Security-Policy` 头作为纵深防御
4. **HTTPS**：外网部署时应使用 HTTPS + `secure` cookie flag
