# 自实验设计：RAG 增强学习 vs 传统阅读学习效果对比

> **实验性质**：自身对照实验（N=1），概念验证级别
> **实验主体**：项目作者自身
> **设计原则**：交叉设计 + 延迟测试，控制知识难度和材料等量

---

## 一、实验目标

**核心问题**：使用 AI 学习助手（RAG 增强学习）是否比传统阅读学习方法带来更好的学习效果？

**具体指标**：
- 即时测评正确率的差异
- 1 小时后知识留存率的差异
- 主观学习体验的差异
- 学习过程中的互动行为差异

---

## 二、实验设计

### 2.1 实验材料选择

选择两个难度相近、对受试者来说均为陌生概念的知识点：

| 参数 | 知识点 A（对照） | 知识点 B（实验） |
|------|------------------|------------------|
| 主题 | 卡尔曼滤波 | 隐马尔可夫模型 |
| 领域 | 信号处理 / 控制理论 | 概率模型 / 机器学习 |
| 预估难度 | 中等 | 中等 |
| 先验知识 | 无 | 无 |
| 教材来源 | 同一教材的不同章节 | 同一教材的不同章节 |
| 材料长度 | 约 3000-4000 字 | 约 3000-4000 字 |

**选择依据**：
1. 两个概念涉及相似难度的数学和逻辑推理
2. 对不具备相关背景的学习者来说都是全新内容
3. 内容量可控，适合 30 分钟的集中学习

**实验前确认**：
- [ ] 确认对两个概念均无任何先验知识
- [ ] 确认两段材料的难度感知一致（阅读引言后主观评分差距 ≤ 1 分）

### 2.2 实验流程

实验分两天进行，每天一个阶段，避免学习疲劳和知识交叉干扰。

## 三、测量指标

### 3.1 核心指标

| 指标 | 定义 | 计算方式 | 预期方向 |
|------|------|----------|----------|
| 即时测评正确率 | 学习后立即测试的正确率 | 即时测试正确题数 / 即时测试总题数 | 实验组 ≥ 对照组 |
| 延迟测试正确率 | 1 小时后测试的正确率 | 延迟测试正确题数 / 延迟测试总题数 | 实验组 > 对照组 |
| 知识留存率 | 延迟测试相对于即时测试的成绩保持比例 | 延迟正确率 / 即时正确率 | 实验组 > 对照组 |
| 主观学习体验 | 对学习过程的主观满意度 | 1-5 分 Likert 量表 | 实验组 > 对照组 |
| 提问次数 | 学习过程中的主动提问次数 | 直接计数（仅实验组有效） | 越多表示越主动 |

### 3.2 辅助指标

| 指标 | 描述 |
|------|------|
| 学习完成度 | 是否完整阅读了所有教材内容 |
| 笔记质量评估 | 对照组笔记的逻辑性、完整性（自评 1-5） |
| 笔记字数 | 对照组笔记的总字数（可选的定量指标） |

## 七、预期结论

### 假设 H1（主要假设）

> **RAG 增强学习在知识留存率上优于传统阅读学习。**

**依据**：
- RAG 交互过程中产生更多的主动回忆和精细加工
- 即时问答打破了阅读的被动接收模式
- 多轮追问促进了深层语义编码

### 假设 H2

> **RAG 增强学习在主观学习体验上优于传统阅读学习。**

**依据**：
- 即时问答减少了"卡住"的挫败感
- 类比解释降低了理解门槛
- 互动性增强了学习趣味

### 假设 H3

> **两组在即时测评正确率上差异不大，但 RAG 组的延迟测试成绩明显更优。**

**依据**：
- 即时测试主要考察短期记忆，两组都能取得较好成绩
- 延迟测试考察长期记忆和深度理解，RAG 组的精细加工效果在此体现

### 替代假设

如果实验结果与预期相反（传统阅读组表现更好），可能的原因包括：
- RAG 交互打断了持续阅读的专注流（context switching cost）
- AI 回复质量不稳定，部分回答可能引入误解
- 30 分钟过短，RAG 交互的优势需要更长的学习时间才能体现

## 八、局限性说明

| 局限 | 说明 | 影响 |
|------|------|------|
| 样本量 = 1 | 仅有实验者自身一个样本 | 结果无法统计检验，不可推广 |
| 单次实验 | 每个条件只进行了一次 | 可能受当天状态（精力、情绪）影响 |
| 知识点不同 | 不同知识点本身难度可能不完全等同 | 即使主观判断难度相近，实际可能存在偏差 |
| 实验者期望效应 | 实验者本人就是被试，知道实验目的 | 可能潜意识中偏向 RAG 组 |
| 顺序效应 | 阶段 A（对照）在前，阶段 B（实验）在后 | 第二天的学习状态可能受第一天影响 |
| 无随机化 | 知识点与实验条件不是随机分配 | 专业知识点的固有差异可能混淆结果 |
| 外部效度低 | 实验室式的短期学习 ≠ 真实的学习场景 | 结论不能直接外推到长期学习效果 |

### 缓解措施

| 局限 | 缓解方法 |
|------|----------|
| 样本量 | 可在不同时间点重复实验 2-3 次，取平均结果 |
| 知识点差异 | 实验前对两个知识点的材料难度主观评分，差异过大则调换分配 |
| 期望效应 | 实事求是记录数据，不在记录时做主观判断 |
| 顺序效应 | 如果重复实验，可交换顺序（先 RAG 后传统）取平均 |
| 知识混淆 | Day 3 的混淆度检查，如果发现明显混淆则结果存疑 |

> **结论**：本实验仅为概念验证（Proof of Concept），结果仅供产品方向参考。如结果符合预期，可作为后续邀请外部用户参与更大规模实验的依据。

## 九、附录：测试题生成说明

为确保测试题目的一致性和客观性，测试题遵循以下原则：

1. **基于教材内容**：所有题目严格基于提供的教材内容，不涉及外部知识
2. **同等难度**：两套测试题的题型配比、难度分布保持一致
3. **延迟测试增量**：延迟测试包含即时测试的全部题目 + 等量新题
4. **涵盖多维度**：从记忆、理解、应用到综合，覆盖布鲁姆认知层次
5. **题目类型**：全部使用客观选择题，避免主观评分的偏差

测试题由第三方 AI（非学习助手系统）生成，确保实验条件不受系统自身偏差影响。
